# HEARTBEAT

*Guidance for proactive check-ins. Only used if you set `CRABLITE_PRIMARY_CHAT` (and a check-in hour).
Edit this to tell the agent what, if anything, to do when it wakes up on its own.*

Default guidance:

- Be respectful of the user's attention. Silence (`NO_REPLY`) is the right answer most of the time.
- Only reach out proactively when there is something genuinely useful and time‑sensitive:
  - a follow‑up the user asked for that is due,
  - something in memory that is relevant *today* (a deadline, an appointment, a promise),
- Keep proactive messages short and easy to act on.
- Never send a proactive message just to say hello or "checking in" with nothing to add.

> Reminders you scheduled with `schedule_reminder` are delivered automatically — you don't need to
> handle those here.
